# EndNote 工作流

## 基本分工

- 写入工作流：用于新增文献、读取记录、修改记录、软删除记录。
- 输出工作流：用于检索文献、读取详情、生成引用、导出 BibTeX、生成 bibliography、辅助生成 TeX。

不要用输出工作流修改 EndNote 图书馆。不要用写入工作流替代检索和引用生成。

## 新增文献

1. 先提取或确认题名、作者、年份、期刊、卷、期、页码、DOI、关键词、摘要和 PDF 路径。
2. 默认生成 EndNote 可导入记录；只有用户明确要求自动化直写时，才进入直写流程。
3. 自动化直写必须先 dry-run，检查待写入记录的题名、作者、年份、期刊、页码、DOI 和引用类型。
4. 期刊文章直写 `.enl` 时必须使用 EndNote 数据库内部类型 `Journal Article -> reference_type 0`。
5. 不要把 EndNote XML 导出中的 `<ref-type>` 数字写入 `.enl`；XML 中 `Journal Article` 常见值是 `17`，它不是 `.enl` 内部写入值。
6. 写入后立即读取新记录，确认作者、期刊、年份、DOI、页码和 `reference_type` 正确。
7. 如果记录类型显示成“法案”等异常类型，先修复 EndNote 源记录或写入 MCP 的类型映射，不要只改 `.bib` 或 `.tex`。
8. 后续如果需要检索、引用或 BibTeX，进入输出同步流程。

## 修改记录

1. 先读取目标记录，确认 record id、题名和作者匹配用户意图。
2. 对比旧值和新值，明确字段级修改计划。
3. 先执行 dry-run。
4. 用户确认后再写入。
5. 写入后重新读取记录，验证修改确实落在目标记录上。
6. 如果发现“未知作者”、乱码作者、缺失期刊、缺失 DOI、年份错误、记录类型错误或 BibTeX 字段不完整，必须修复 EndNote 源记录或最新导出数据，不要只修改生成的 `.bib` 或 `.tex`。

## 删除记录

1. 先读取目标记录，确认 record id、题名和作者。
2. 只按明确 record id 删除。
3. 先执行 dry-run。
4. 用户确认后执行软删除。
5. 删除后重新读取或检索，验证记录状态。

不要硬删除数据库行。不要在没有明确目标列表时批量删除。

## 输出同步

1. 确认 EndNote 中的新增、修改或删除已经完成。
2. 使用最新 EndNote 导出数据重建输出索引。
3. 重建后再检索、读取详情、生成引用、导出 BibTeX 或生成 TeX。

任何写入工作流完成后，都要先同步输出层，再相信检索和引用结果。

## 检索和引用

1. 先按用户给出的关键词、作者、题名、年份、DOI 或主题检索。
2. 读取候选文献详情，确认目标文献。
3. 根据用户需要生成引用、bibliography、BibTeX 或 TeX。
4. 如果输出字段错误，回到修改记录流程，修复 EndNote 源记录后重新同步。

## TeX 输出

1. 从已确认的 EndNote 记录生成 BibTeX。
2. 让 `.tex` 中的 cite key 与 `.bib` 条目一致。
3. 中文内容优先使用 XeLaTeX。
4. 如果引用 undefined，检查 cite key、`.bib` 路径和构建日志。
5. 如果 BibTeX 元数据错误，修复 EndNote 记录后重新同步输出。
